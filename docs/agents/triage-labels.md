# Triage Labels

The engineering skills speak in canonical triage roles. This repo currently
uses GitHub's default labels plus `V` for Varad/frontend/editor work. The
triage state labels below can be created when active triage begins.

| Skill role | GitHub label | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate this issue |
| `needs-info` | `needs-info` | Waiting on reporter for more information |
| `ready-for-agent` | `ready-for-agent` | Fully specified, ready for an AFK agent |
| `ready-for-human` | `ready-for-human` | Requires human implementation |
| `wontfix` | `wontfix` | Will not be actioned |

For the current editor/AI implementation track, keep `V` on all related issues.
